1 Prerequisites

Ensure you have the following installed on your machine:

  • Python 3.11+ — python3 --version
  • pip & venv — usually bundled with Python
  • Git — for cloning the repository
  • A Telegram account — for 2FA approval of transactions
  • RPC endpoints — Infura, Alchemy, or a local node for your target chains
💡
For quick testing, use the Sepolia testnet. You can get free test ETH from sepoliafaucet.com.

2 Clone & Install

Terminal
$ git clone https://github.com/anthropic/clawsafe-pay.git
$ cd clawsafe-pay
$ python3 -m venv .venv
$ source .venv/bin/activate

Install dependencies for each service:

Terminal
$ pip install -r publisher_service/requirements.txt
$ pip install -r signer_service/requirements.txt
$ pip install -r user_auth/requirements.txt
$ pip install -r transaction_builder/requirements.txt

3 Environment Configuration

Create a .env file in the project root. Here's the template:

.env
# ── Chains ─────────────────────────────────────────
SEPOLIA_RPC_URL=https://sepolia.infura.io/v3/YOUR_KEY
BASE_RPC_URL=https://mainnet.base.org

# ── Signer Service ─────────────────────────────────
SIGNER_PRIVATE_KEY=0xYOUR_PRIVATE_KEY
SIGNER_HMAC_SECRET=your-shared-hmac-secret
SIGNER_PORT=8001

# ── Publisher Service ──────────────────────────────
PUBLISHER_HMAC_SECRET=your-shared-hmac-secret
PUBLISHER_PORT=8002

# ── User Auth (Telegram) ──────────────────────────
TELEGRAM_BOT_TOKEN=123456:ABC-your-bot-token
TELEGRAM_CHAT_ID=your-chat-id
USER_AUTH_PORT=8000

# ── Optional ──────────────────────────────────────
REVIEWER_ENABLED=false
REVIEWER_MODEL=claude-sonnet-4-20250514
🔐
Never commit your .env file. It contains private keys and secrets. The .gitignore already excludes it.

4 Launch Services

Start each service in a separate terminal (or use the demo script):

Terminal 1 — User Auth
$ python -m user_auth.main
# → User Auth running on :8000
Terminal 2 — Signer
$ python -m signer_service.main
# → Signer Service running on :8001
Terminal 3 — Publisher
$ python -m publisher_service.main
# → Publisher API running on :8002

Or use the all-in-one demo launcher:

Terminal
$ bash demo.sh
✅
Once running, visit http://localhost:8002 for the homepage and http://localhost:8002/demo for the live dashboard.

5 Telegram Bot Setup

The human-in-the-loop 2FA requires a Telegram bot:

  1. Open Telegram and message @BotFather
  2. Send /newbot and follow the prompts to create your bot
  3. Copy the bot token into TELEGRAM_BOT_TOKEN in your .env
  4. Start a conversation with your bot and send any message
  5. Get your chat ID: curl https://api.telegram.org/bot<TOKEN>/getUpdates
  6. Set TELEGRAM_CHAT_ID in your .env
⚠️
Only add your own Telegram chat ID. The bot will send transaction approval requests to this chat. Anyone with access can approve or reject transactions.

6 Your First Transaction

Submit a test payment intent via the API:

cURL
$ curl -X POST http://localhost:8002/intents \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{
    "chain": "sepolia",
    "to": "0x742d35Cc6634C0532925a3b844Bc9e7595f...",
    "amount": "0.001",
    "asset": "ETH",
    "agent_id": "test-agent"
  }'

The flow: Publisher validates → builds draft transaction → sends to Signer → Signer requests Telegram approval → on Approve, signs & broadcasts → returns tx hash.

7 Adding Chains

ClawSafe Pay supports multiple chains through its registry system. Each chain module lives in chains/:

chains/
chains/
├── evm/
│   ├── sepolia/    # Ethereum Sepolia testnet
│   └── base_l2/    # Base L2 mainnet
├── solana/         # Solana mainnet/devnet
├── bitcoin/        # Bitcoin mainnet/testnet
├── cardano/        # Cardano mainnet
├── zcash/          # Zcash mainnet
└── tokens/         # USDC, USDT token support

To add a new EVM chain, create a folder in chains/evm/ with a __init__.py that registers the chain config, provider, builder, and signer. The chain is then available for use in payment intents.

🧩
Browse pre-built chain modules on ClawHub — install new chains with a single command.

8 Production Deployment

For production, follow these additional steps:

  • Air-gap the signer: Run the signer service on a separate, firewalled machine that only accepts connections from the publisher
  • Use a hardware wallet: For maximum security, integrate with a Ledger or Trezor instead of a software key
  • Enable the reviewer: Set REVIEWER_ENABLED=true to add an LLM risk analysis step for every transaction
  • Set spend limits: Configure per-agent daily and per-transaction limits in the policy engine
  • Enable HTTPS: Put services behind a reverse proxy (nginx, Caddy) with TLS termination
  • Rotate secrets: Regularly rotate HMAC secrets and API keys
🔐
Never expose the signer service to the public internet. It should only be reachable from the publisher service on your internal network.
