1 Prerequisites

Ensure you have the following installed on your machine:

  • Python 3.11+ — python3 --version
  • pip & venv — usually bundled with Python
  • Git — for cloning the repository
  • Docker + Docker Compose — optional, for one-command local runs
  • A Telegram account — for 2FA approval of transactions
  • A funded Sepolia wallet — configure via WALLET_ADDR_1 and WALLET_PRIV_KEY_1
  • RPC endpoint — set SEPOLIA_RPC_URL (public, Infura, Alchemy, or your own node)
💡
For quick testing, use the Sepolia testnet. You can get free test ETH from sepoliafaucet.com.

2 Clone & Install

Terminal
$ git clone https://github.com/anthropic/clawsafe-pay.git
$ cd clawsafe-pay
$ python3 -m venv .venv
$ source .venv/bin/activate

Install dependencies for backend services, reviewer, and frontend:

Terminal
$ pip install -r backend/publisher_service/requirements.txt \
  -r backend/signer_service/requirements.txt \
  -r backend/user_auth/requirements.txt \
  -r reviewer_service/requirements.txt \
  -r frontend/requirements.txt

Prefer a containerized setup? Use Docker Compose instead:

Terminal
$ cp .env.example .env
$ docker compose up --build

3 Environment Configuration

Create your environment file from the repo template, then edit required values:

Terminal
$ cp .env.example .env

Minimum values you should set in .env:

.env
# ── User Auth + shared security ───────────────────
TELEGRAM_BOT_TOKEN=123456:ABC-your-bot-token
TELEGRAM_CHAT_ID=your-chat-id
HMAC_SECRET=replace-with-random-secret
AUTH_SERVICE_PORT=8000
SIGNER_SERVICE_CALLBACK_URL=http://localhost:8001/auth/callback

# ── Signer Service ─────────────────────────────────
SIGNER_SERVICE_PORT=8001
USER_AUTH_URL=http://localhost:8000
WALLET_ADDR_1=0xYourSepoliaWalletAddress
WALLET_PRIV_KEY_1=0xYourSepoliaPrivateKey

# ── Publisher Service ──────────────────────────────
PUBLISHER_SERVICE_PORT=8002
PUBLISHER_API_KEY=change-me-publisher-key
SIGNER_SERVICE_URL=http://localhost:8001
SIGNER_FROM_ADDRESS=0xYourSepoliaWalletAddress
SEPOLIA_RPC_URL=https://rpc.sepolia.org
POLICY_RECIPIENT_ALLOWLIST=*

# ── Frontend + Reviewer (optional) ───────────────
DASHBOARD_PORT=8008
REVIEWER_SERVICE_URL=http://localhost:8003
REVIEWER_SERVICE_PORT=8003
🔐
Never commit your .env file. It contains private keys, API keys, and shared secrets. The project .gitignore already excludes it.

4 Launch Services

For manual startup, export PYTHONPATH once so backend modules resolve correctly:

Terminal
$ export PYTHONPATH=backend:$PYTHONPATH

Then start each service in a separate terminal:

Terminal 1 — User Auth
$ python -m user_auth.main
# → user_auth on :8000
Terminal 2 — Signer
$ python -m signer_service.main
# → signer_service on :8001
Terminal 3 — Publisher
$ python -m publisher_service.main
# → publisher_service on :8002
Terminal 4 — Frontend
$ python -m frontend.main
# → frontend dashboard/home on :8008
Terminal 5 — Reviewer (optional)
$ python -m reviewer_service.main
# → reviewer_service on :8003

Or use the all-in-one demo launcher:

Terminal
$ bash demo.sh
$ bash demo.sh stop

Container alternative:

Terminal
$ docker compose up --build
✅
Once running, open http://localhost:8008 (homepage/setup/security) and http://localhost:8008/demo (interactive dashboard). The publisher API is at http://localhost:8002.

5 Telegram Bot Setup

The human-in-the-loop 2FA requires a Telegram bot:

  1. Open Telegram and message @BotFather
  2. Send /newbot and follow the prompts to create your bot
  3. Copy the bot token into TELEGRAM_BOT_TOKEN in your .env
  4. Start a conversation with your bot and send any message
  5. Get your chat ID: curl https://api.telegram.org/bot<TOKEN>/getUpdates
  6. Set TELEGRAM_CHAT_ID in your .env
⚠️
Only use your own chat ID (or a tightly controlled ops group). Anyone who can press the Telegram approval button can authorize a real transaction.

6 Your First Transaction

Submit a test payment intent via publisher_service:

cURL
$ curl -X POST http://localhost:8002/intent \
  -H "Content-Type: application/json" \
  -H "X-API-Key: change-me-publisher-key" \
  -d '{
    "intent_id": "pay-001",
    "from_user": "alice",
    "to_user": "bob",
    "chain": "sepolia",
    "asset": "ETH",
    "amount_wei": "1000000000000000",
    "to_address": "0xd8da6bf26964af9d7eed9e03e53415d37aa96045",
    "note": "first payment test"
  }'

Poll status after approving in Telegram:

cURL
$ curl -X GET http://localhost:8002/intent/pay-001 \
  -H "X-API-Key: change-me-publisher-key"

Flow summary: publisher_service validates intent → builds draft transaction → calls reviewer_service → submits to signer_service → user_auth requests Telegram approval → signer broadcasts and returns tx hash.

7 Adding Chains

ClawSafe Pay uses a registry-based chain architecture in backend/chains/:

backend/chains/
backend/chains/
├── evm/
│   ├── sepolia/    # Ethereum Sepolia testnet
│   └── base_l2/    # Base L2 mainnet
├── solana/         # Solana family modules
├── bitcoin/        # Bitcoin family modules
├── cardano/        # Cardano family modules
├── zcash/          # Zcash family modules
└── tokens/         # USDC, USDT token support

To add a new chain, create a module package with config/provider/builder/signer and call register_chain(...) in that package's __init__.py. Then import the package from backend/chains/__init__.py so it auto-registers at startup.

🧩
Browse pre-built chain modules on ClawHub — install new chains with a single command.

8 Production Deployment

For production, follow these additional steps:

  • Isolate signer network access: Keep signer_service on a private network reachable only by publisher_service
  • Protect signing keys: Store WALLET_PRIV_KEY_N in a secret manager and rotate regularly
  • Rotate auth secrets: Rotate HMAC_SECRET, PUBLISHER_API_KEY, and any agent keys on a schedule
  • Enforce policy limits: Set strict POLICY_MAX_AMOUNT_WEI and POLICY_RECIPIENT_ALLOWLIST
  • Enable reviewer hardening: Configure REVIEWER_SERVICE_URL and supply ZAI_API_KEY for LLM risk analysis
  • Use TLS and ingress controls: Place public endpoints behind HTTPS + IP filtering / WAF
  • Prefer container orchestration: Run with docker compose or your platform's equivalent for restart/health policies
🔐
Never expose the signer service to the public internet. It should only be reachable from the publisher service on your internal network.
